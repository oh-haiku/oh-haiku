1 Introduction

1.1 Purpose of application

The germ of this application is a tweet. The Swedish comedian Jesper RÃ¶nndahl (@jesperronndahl) tweeted that he often thinks in chunks of three unrelated thoughts. He indicated that he would like to have an app that could tell him whether these three sentences constitute a haiku poem.

1.2 General characteristics of application

This application is a simple and easy-to-use poem-checker. It can be used to verify that a piece of text is written according to Haiku-rules and tweet the text. The application is also an easy way to store the users poems.

2 Test environment

First, read Developer manual to get the code or .apk and install.

To do code coverage you need to install ant and an easy way to do that is using brew. After installing ant you could easily write Apache ant-commands in Terminal. Check more precise commands here. You HAVE TO run the emulator before code coverage.

To update the project to the new development environment, go to Managing projects from the command line and Testing from other IDEs. The android tool will generate any files and folders that are either missing or need to be updated, as needed for the Android project. In Terminal, type: android update project –name Ohhaiku –target 2 –path android update test-project -m ../Ohhaiku -p

To get the coverage, navigate to /Ohhaikutest and write: ant emma debug install test 

open coverage/coverage.html

To install the .apk file on an emulator, follow this guide: (http://blog.freewarelovers.com/2010/08/how-to-install-apk-files-on-android.html) To test Twitter capabilites, Internet access is required.
